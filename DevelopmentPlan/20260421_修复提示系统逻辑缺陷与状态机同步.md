# 修复提示系统逻辑缺陷与状态机同步

## 任务描述
修复无书籍播放静默无提示、彻底根除 ErrorListener 导致的 UI 闪烁、将报错职责从提词器剥离并上抛给吉祥物。

## 详细步骤

### 1. 修复无书籍播放静默漏洞 (CyberPlayerConsole)
- 检查播放按钮回调，如果 `ReaderProvider.toggleTTS()` 返回 `TtsToggleResult.noContent`，强制调用 `TtsEngineService.setLastError('无法开启 TTS：请先导入书籍')`，以触发全局错误通知。

### 2. 根除 UI 闪烁 Bug (TtsErrorListener)
- 彻底移除 `tts.clearLastError()` 及其对应方法。
- 说明：Listener 作为状态观察者，不应擅自清空状态，防重弹逻辑应仅依赖本地 `_previousError`。

### 3. 提词器职责剥离 (TeleprompterView)
- 移除内部对 `tts.lastError` 的直接文本渲染。发生错误时，保持"暂停"或"缓冲"的正常视觉表现即可，错误提示全部交给全局 Toast 与吉祥物。

### 4. 吉祥物 (Xiaoyo) 错误联动预埋 (BoardMascot)
- 让 `BoardMascot` 监听 `TtsEngineService`，若 `tts.lastError != null`，切换表情至警告或宕机状态（如眼球变红/闭眼等）。

### 5. 状态机自动恢复 (TtsEngineService)
- 在 `_downloadTtsAudio` 成功获取 `filePath` 以及 `_speakWithLocalTts` 成功后，显式调用 `_clearLastError()` 实现自动状态恢复。

### 6. 文档同步与提交
- 更新 `development_log.md` 记录修改细节，并提交代码。

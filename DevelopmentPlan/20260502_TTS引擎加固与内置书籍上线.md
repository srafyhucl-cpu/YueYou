- [x] **暂停跳句修复**：暂停恢复后不再自动跳过当前句。
- [x] **提词器物理同步**：KTV 扫光动画由底层音频播放头驱动，实现毫秒级同步。

## 🏗 核心修复与实现

### 一、TTS 引擎加固

- **`_playCompleter` 信号**：在 `TtsEngineService.playFile` 中引入信号量，`stop()` 时立即解除阻塞，彻底消除 30s 超时等待。
- **`refreshSession` 原子化**：先 `_session++` + `_buffer.clear()`，再 `await stopAudio()`，切换状态清理前置，防止旧回调干扰新会话。
- **`_isPausing` 保护锁**：暂停期间屏蔽 `_onPlaybackComplete` 的进度推进，修复暂停跳句问题。
- **`resetFetchIndex` 接口**：`TtsSentenceSource` 新增该方法，切换时强制同步预取游标与可见游标，消除切换后"跳过一两句"。
- **双重会话校验**：`BufferedAudio` 增加 `session` 标识，过期音频精准丢弃。

### 二、提词器同步优化

- **物理进度驱动**：`TeleprompterView` 改由 `AudioPlayer.onPositionChanged` 物理流驱动 KTV 扫光，彻底废弃字数估算算法。
- **渲染解耦**：滚动逻辑从 `build()` 移至 `StreamSubscription`，消除副作用，提升 120Hz 高刷性能。

### 三、设置页 UI 重构

- **静默暂停**：`TtsAudioNotifier` 实现 `_idleTimer`，`main.dart` 全局 `Listener` 捕获触控心跳，支持 1-60 分钟自动停播。
- **氛围音风格**：新增"江湖风云（武侠）"与"围炉夜话（温馨）"两种风格，算法调整粉噪声参数，无需外部音频资源。
- **选项精简**：移除冗余语速选项，自定义 `_ChoiceSelector` 替代原生下拉框。

### 四、内置默认书籍（西游记）

- **`DefaultBookService`**：三级降级（本地缓存 → 网络 → 内置常量），分离下载原则，并发去重 + 影子预读。
- **`ReaderProvider` 跨章状态机**：`ChapterLoadState` 枚举 + `loadChapter` + `_autoAdvanceChapter`，章末自动推进。
- **`BookshelfProvider` 启动链**：新用户书架为空时自动注入默认书籍，粘性位防重入。
- **UI 反馈**：`TeleprompterView` 消费 `chapterLoadState`，展示"正在同步云端卷宗..."与错误重试浮层。

### 五、Go 服务端上线

- **多文件模块化**：`server/` 目录统一管理（`main.go` / `config.go` / `handler_tts.go` / `handler_book.go`）。
- **书籍接口**：`GET /api/v1/book/catalog` 返回 embed 嵌入的目录 JSON；`POST /api/v1/book/chapter` 按 chapterIndex 派发 OSS CDN 地址。
- **数据转换**：`books/convert_to_oss.py` 批量将 `xiyouji.json` 转为 100 个 UTF-8 章节 txt，上传至 OSS。
- **生产部署**：systemd 托管，密钥通过 `EnvironmentFile` 注入，AK 不入版本库，全链路联调验证通过。

## ⚠️ 风险提示

- **索引对齐**：`BookConstants` 标题索引必须与 OSS 文件命名（`001.txt` = index 0）严格一致。
- **GPU 负载**：提词器高频刷新区域必须用 `RepaintBoundary` 隔离。
- **内存安全**：MP3 临时文件遵循"阅后即焚"，在 `_deleteFile` 中及时回收。
- **AK 安全**：服务端 AccessKey 只通过环境变量注入，严禁写入代码或版本库。

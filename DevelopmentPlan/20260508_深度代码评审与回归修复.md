# 深度代码评审与回归修复

**日期**: 2026-05-08 | **范围**: Flutter 客户端（TTS / 阅读器 / 2048 / 书库 / CI 门禁）

## 背景

针对 P0 / P1 / P2 三档严重度对全栈做了一次深度审计，并以"小步快跑、每修一项配一条回归用例"的方式逐项落地。本次共计修复 P0 × 8、P1 × 6 项，新增/更新 12 条针对性回归用例，全量 484 用例零失败、零警告。

## P0 修复（必须当版发版前合入）

### P0-1 ReaderProvider build 副作用消除

- 把 `_handleErrorState` 从 build/state listener 同步路径移到 `WidgetsBinding.instance.addPostFrameCallback`，杜绝 build 期间触发 setState 引发的 dirty marker 异常。
- 文件：`lib/features/reader/providers/reader_provider.dart`

### P0-2 StorageService 防 NPE

- `StorageService._p` 在未初始化时改抛 `StateError('StorageService 未初始化，请先调用 init()')`，替代隐式空指针 → 让早期违规调用立刻失败定位。
- 文件：`lib/core/database/storage_service.dart`

### P0-3 main.dart 生命周期 hidden→pause

- AppLifecycleState.hidden 不再被错误吞掉，与 paused 走同一释放分支，避免后台进程仍持有音频/Wakelock。
- 文件：`lib/main.dart`

### P0-4 暂停时 mp3 文件不得"阅后即焚"

- `_playNext` 在 `await playFile` 返回后无条件 `_deleteFile`，导致 pause→resume 时 `_buffer.prepend` 的文件已不存在 → playFile 因文件缺失立即 onComplete → 推进游标 → **用户感知"暂停后跳一句"**。
- 修复：用 `_currentFilePath == item.filePath` 作为"暂停中断"耐久标识；命中即保留文件供 resume 复用，自然完成路径才删除。
- 文件：`lib/features/audio/providers/tts_audio_notifier.dart`
- 回归：`test/features/audio/tts_audio_notifier_test.dart` → "暂停后保留当前 mp3 文件供 resume 复用，杜绝跳句"

### P0-5 nextTtsSentence 取模回卷漏洞

- `_fetchIndex = (cursor + 1) % _sentences.length` 在章末仅剩噪音行时会绕回 0，导致 **重复朗读章首句**。
- 修复：去掉取模，越界一律钉到 `_sentences.length`，下次扫描早返回。
- 文件：`lib/features/reader/providers/reader_provider.dart`
- 回归：`test/features/reader/reader_provider_test.dart` → "章末连续噪音行" + "全文皆噪音" 共 2 条用例。

### P0-6 TtsAudioBuffer 排序漏洞

- `add()` 内部按 lineIndex 升序排序的隐性约定与 prepend(...) 语义冲突 → resume 时插队失效。
- 修复：去掉自动 sort，改为"调用方决定顺序"，prepend 走头部、add 走尾部。
- 文件：`lib/features/audio/domain/tts_audio_buffer.dart`
- 回归：新建 `test/features/audio/tts_audio_buffer_test.dart` 8 条用例覆盖 prepend/add/clear/边界。

### P0-7 AndroidManifest cleartext 关闭 + NetworkSecurityConfig 白名单

- `usesCleartextTraffic` 删除 + `network_security_config.xml` 仅放行调试 host（10.0.2.2 / localhost / 127.0.0.1），生产域名一律 https。
- 文件：`android/app/src/main/AndroidManifest.xml`、`android/app/src/main/res/xml/network_security_config.xml`

### P0-8 TtsConfig 默认 server 改 localhost + CI 扫描

- `TtsConfig.serverUrl` 默认值改回 localhost；同步在 `scripts/ai_checks/rules.dart` 新增 `ProductionDomainDefaultRule`，扫描全仓 `String.fromEnvironment(...).defaultValue`，凡是非 localhost / 127.0.0.1 / 10.0.2.2 的 http(s) URL 一律 BLOCKING。
- 营销/合规链接走显式白名单（`PRIVACY_POLICY_URL`、`MARKET_DOWNLOAD_URL`），新增条目需在 PR 描述中说明合规理由。
- 文件：`lib/core/config/tts_config.dart`、`scripts/ai_checks/rules.dart`

## P1 修复

### P1-1 Wakelock 重构

- `playFile` 在每句结束调 `_syncWakeLock(false)`；下一句进入相同 playing 状态时 `syncShadow` 因 `_state` 未变化不再申请 → **从第二句起屏幕熄灭**。
- 修复：删除 playFile 内的 disable 调用，wakelock 完全由状态机驱动（playing/buffering→ON，stopAll/pause/dispose→OFF）。
- 文件：`lib/features/audio/services/tts_engine_service.dart`
- 回归：`test/features/audio/tts_engine_service_test.dart` → "连读两句之间 playFile 不得调用 wakelock.disable（P1-1）"

### P1-2 eliminateTileById 必须重算分数

- 全盘求和模式下，旧实现仅清空 tile + 改写 `_isOver`，未调 `updateScore()` → UI 显示虚高分数。
- 修复：消除后调 `updateScore()` + 显式 `_isOver = !_movesAvailable()`。
- 文件：`lib/features/game_2048/providers/game_provider.dart`
- 回归：`test/features/game_2048/game_provider_test.dart` 新增 "score 必须刷新为剩余 tile 总和" + "GameOver 状态下消除一格仍无路则保持 isOver" 共 2 条。

### P1-3 默认书《西游记》标题特例

- 默认书 `currentBookId='xiyouji'` 而 `BookModel.id=999`，比对永远失败 → 控播台显示"阅游"占位。
- 修复：抽出顶层纯函数 `resolveNovelTitle(String? bookId, List<BookModel> shelf)`，默认书 key 命中即返回 `BookConstants.defaultBookTitle`。
- 文件：`lib/features/audio/presentation/widgets/cyber_player_console.dart`
- 回归：新建 `test/features/audio/cyber_player_console_test.dart` 5 条用例。

### P1-4 滑动音效迁到 if(moved) 内

- 旧实现在 `move()` 入口处依据"盘上有 tile>2"就触发触觉反馈，**无效滑动也响**。
- 修复：删除入口预触发块；改在 `if (moved)` 内分两支处理 —— 有合并走 `playMerge`（内部已含触觉），无合并补一次 `playMoveFeedback`。
- 文件：`lib/features/game_2048/providers/game_provider.dart`
- 回归：`test/features/game_2048/game_provider_test.dart` 新增 "无效滑动绝不触发" + "有效但无合并仅触发一次" 共 2 条。

### P1-5 LibraryScreen.dispose 取消导入

- 已通过 `_CyberImportButtonState.dispose()` 调 `FileImportService.cancelImport()` 实现；本次补幂等性回归用例。
- 文件：`test/features/library/file_import_service_test.dart`

### P1-6 _RealHttpClient.download 改流式

- 原实现 `bytes.addAll(chunk)` 累积全部字节再 `writeAsBytes` → 数百 KB ~ 数 MB mp3 瞬时占用 2x 内存。
- 修复：改为 `IOSink.openWrite()` 边读边写；任何异常均删除半成品文件，避免损坏缓存被 playFile 误识。
- 文件：`lib/features/audio/services/tts_engine_service.dart`

## 工程门禁规则升级

- 命中非 localhost/127.0.0.1/10.0.2.2 的 host → BLOCKING。
- 营销/合规白名单 `_allowedMarketingEnvNames = { 'PRIVACY_POLICY_URL', 'MARKET_DOWNLOAD_URL' }`。

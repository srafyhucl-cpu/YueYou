# 任务记录：TTS 引擎加固与提词器同步优化 (2026-05-02)

## 1. 任务背景
针对用户反馈的 TTS 切换延迟、异常降级、提词器进度错位等核心交互痛点进行专项治理。

## 2. 核心修复与优化 (已完成)

### 2.1 TTS 切换延迟与降级修复
- **[修复] 切换即时响应**：通过 `_playCompleter` 信号强制中断异步阻塞，实现毫秒级停播响应。
- **[修复] 降级状态污染**：在 `refreshSession` 中原子化重置错误计数器与会话 ID，彻底杜绝切换导致的异常降级。
- **[修复] 暂停跳句**：引入 `_isPausing` 保护锁，防止因手动暂停误触发进度推进逻辑。

### 2.2 赛博提词器 (KTV View) 同步优化
- **[优化] 物理进度驱动**：重写 `TtsEngineService` 与 `TeleprompterView` 的交互逻辑，由底层音频播放头（onPositionChanged）驱动，实现 100% 同步。
- **[重构] 渲染解耦**：将滚动逻辑从 `build` 阶段移至 `StreamSubscription` 监听层，消除副作用并提升 120Hz 高刷屏下的性能表现。

### 2.3 设置页面 UI 与功能升级
- **[新增] 静默暂停**：实现全局“省电管理”逻辑，结合全局交互监听器，支持 1m-60m 自动停播。
- **[新增] 氛围音意境**：更新为“江湖风云 (武侠)”与“围炉夜话 (温馨)”，提升沉浸感。
- **[优化] 交互质感**：重构选项选择器组件，增加渐变高亮、阴影浮雕及动效反馈。
- **[精简] 选项精简**：移除冗余的语速调节选项，降低用户认知负担。

## 3. 验证情况
- **自动化验证**：通过 `quick_fix_verify_test.dart` 和 `teleprompter_sync_verify_test.dart` 脚本，验证了 Session 隔离、物理进度同步及零时长保护等关键路径。

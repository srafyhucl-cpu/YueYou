### PR-3 TTS 数据安全与服务端防滥用

**目标**：消除原文日志、公开永久音频和匿名无限调用风险。

**实施项**：

- [x] 删除当前生产权威 Go 服务端朗读原文日志；未跟踪 Python 服务不进入生产权威。
- [x] 日志只记录不可逆请求 ID、字符数、音色、耗时、状态码和错误分类。
- [ ] TTS 对象统一写入私有 Bucket。
- [x] 对象键改为服务端密钥参与的 HMAC-SHA256 或随机 ID，禁止裸 MD5 文本指纹。
- [x] 返回有效期不超过 10 分钟的签名下载 URL。
- [ ] OSS 生命周期规则自动删除 24 小时以上的临时 TTS 音频。
- [x] 单请求正文上限保持 2000 字，同时增加 HTTP 请求体字节上限。
- [x] 增加 IP 和匿名安装 ID 双维度限流、全局并发上限和有界等待队列。
- [x] 请求取消后必须退出信号量等待或合成流程。
- [x] 增加 `/health` 和 `/ready`，分别表示进程存活与 OSS/TTS 依赖可用。
- [x] 更新隐私政策，明确处理目的、第三方、音频保留期限和删除方式。

**建议初始阈值**：

| 项目 | 初始值 | 调整依据 |
| --- | --- | --- |
| 单 IP 请求 | 30 次/分钟 | 生产监控与误伤率 |
| 单安装 ID 请求 | 120 次/小时 | 连续听读的句子吞吐 |
| 全局合成并发 | 1 | edge-tts 当前约束 |
| 等待队列 | 50 | 防止无界 goroutine/协程 |
| 请求体上限 | 16KB | 2000 中文字符及 JSON 余量 |
| 签名 URL 有效期 | 10 分钟 | 下载重试需求 |
| OSS 临时对象生命周期 | 24 小时 | 本地缓存已承担长期复用 |

**验收**：

- 日志、Sentry 和错误响应均不包含用户原文。
- 未签名 URL 无法访问音频；过期 URL 返回拒绝。
- 超限请求返回 `429`，队列满时快速失败。
- 并发、取消、超时、OSS 失败和签名过期均有自动化测试。

**2026-07-12 执行记录**：

- `server/handler_tts.go` 移除明文原文日志与裸 MD5 对象键；TTS 临时对象键改为
  `HMAC-SHA256(voice + text)`，对象路径为 `cache/tts/v2/*.mp3`。
- 上传 TTS 音频时显式使用 `oss.ObjectACL(oss.ACLPrivate)`，返回 URL 改为
  OSS `SignURL` 生成的 10 分钟短效下载链接；生产 Bucket 级私有 ACL 与 24 小时

- 服务端新增 16KB HTTP 请求体上限、2000 字符正文上限、IP 每分钟 30 次和匿名安装
  ID 每小时 120 次限流、全局 edge-tts 并发 1、等待队列 50，以及取消等待退出逻辑。

  `X-YueYou-Install-ID`；该 ID 仅用于服务端限流，不包含阅读内容或设备信息。
- 新增 `/health` 和 `/ready`；`/ready` 同时检查 OSS 存储与 `edge-tts` 执行器。
- `server/handler_privacy.go` 更新 TTS 处理目的、日志脱敏、签名 URL 有效期、
  临时音频保留目标和第三方服务说明。

## 8. M3：听读产品主线验证

### 8.1 产品定位

主定位固定为“沉浸式本地小说听读伴侣”。2048 是听书时的可选陪伴模式，不再与阅读并列扩张为第二套产品。

### 8.2 优先能力

- [ ] 首次导入到开始朗读不超过 3 步。
- [ ] 睡眠定时、章节跳转、书签、历史位置和离线缓存稳定可恢复。
- [ ] TTS 弱网降级不重复朗读、不跳句、不丢进度。
- [ ] 游戏模式可完全隐藏，纯阅读用户不受干扰。
- [ ] 辅助功能覆盖字体缩放、屏幕阅读器标签、对比度和触控目标。
- [ ] 选择 3 档清晰动画质量，移除无法用真实帧数据证明价值的同步 CPU 跑分。

### 8.3 验证方式

- 招募至少 5 名目标用户完成“导入一本书并听 10 分钟”的可用性测试。
- 全程不采集阅读正文；产品数据优先使用现场观察和用户主动反馈。
- 如引入匿名遥测，必须单独征得同意，只记录非内容事件，并更新隐私政策。

### 8.4 产品决策门 G1

以下条件全部满足才进入 AI 阶段：

- [ ] 5 名用户中至少 4 名无需指导完成导入和朗读。
- [ ] 10 分钟听读中无重复句、跳句和不可恢复中断。
- [ ] 至少 3 名用户明确愿意再次使用。
- [ ] P0/P1 发布问题保持关闭，连续两周 CI 稳定通过。

未通过时继续修复核心体验，不启动 AI 功能。

## 9. M4：AI 试点

### 9.1 唯一 MVP

首个 AI 能力只做“带章节引用的西游记上下文问答”，暂不同时做角色扮演、续写、摘要和 2048 策略。

**范围**：

- 用户针对当前章节或全书提问。
- 回答必须附章节号和可跳转原文片段。
- 检索无可靠证据时明确回答“未找到依据”。
- 会话历史设置长度与 Token 上限，不永久保存用户问题。
- AI 服务作为独立 Python 服务，通过内部 API 与 Go 网关协作。

### 9.2 AI 决策门 G2

- [ ] 预设事实题准确率不低于 85%。
- [ ] 引用章节命中率不低于 90%。
- [ ] P95 响应时间低于 5 秒。
- [ ] 单次问答成本和月度预算有硬上限。
- [ ] 5 名测试用户中至少 3 名认为问答提升了阅读体验。

未达标时停止扩展，不继续增加角色聊天和内容生成功能。

## 14. 暂停与回滚条件

- 签名升级链不明确时，暂停更换 JKS，先确认应用市场规则。
- 隐私闸门导致启动死循环或无法恢复时，回滚该 PR，不允许临时恢复同意前网络初始化。
- TTS 私有 URL 导致播放失败时，回滚下载实现，不回滚原文脱敏和限流。
- 架构拆分引入生命周期竞态、循环依赖或测试行为变化时，整 PR 回滚并重新划分职责。
- AI 准确率、引用率、成本或用户价值未达到 G2 时停止扩展，保留稳定听读主线。

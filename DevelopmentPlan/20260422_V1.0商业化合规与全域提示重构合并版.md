# 2026-04-22 V1.0 商业化合规与全域提示系统重构总览

此文档为今日内创建的四个独立重构任务的合并归档。

## 🎯 阶段一：商业化纯净度提亮及全域合规构建

**目标**：清剿代码债务，达成极客核心架构规范，确保产品能随时发版。

- **设计体系全局合规**：深入 UI 组件底层，彻底消灭「魔法数字」布局，将所有边距和圆角收口为 `CyberDimensions` 定义的尺寸字典。
- **全局错漏与报警统一**：新建常量文案库 `lib/core/constants/cyber_error_messages.dart`，替换所有零散字符串。
- **架构红线验证**：确保 `domain` 层无 UI 库污染，高频画布区域有效 `RepaintBoundary` 隔离，文本大文件解析 `Isolate` 异步化，遵守 platform 分离下载规范。
- **CI 验证**：达成 `flutter analyze` 0 Error, 0 Warnings, 0 Issues。顺利通过 `flutter build apk --release`。

## 🎯 阶段二：全域提示系统与错误链路重构

**目标**：优化全域视觉提示反馈机制，彻底解决报错乱弹现象。

- **CyberToast 视觉升级**：改用动态安全区适配高度；加入 `BackdropFilter` 毛玻璃效果 (`sigma: 10`)；支持长文本换行；应用单例 Overlay 管理防堆叠。
- **TtsEngineService 业务信息映射**：收口异常类与网络状态码进行中文化降级提示脱敏，彻底避免暴露技术堆栈给终端用户。

## 🎯 阶段三：提示系统逻辑漏洞与状态同步修复

**目标**：解决部分遗漏逻辑冲突，根除界面闪烁现象，联动吉祥物表现。

- **播放静默修复**：当无书籍触发 `toggleTTS` 时，拦截 `noContent` 状态并弹出无书籍提示。
- **TtsErrorListener 去权与防抖**：移除 `clearLastError` 的非法调用（状态观察者不应修改状态），利用本地 1000ms 防抖机制消除多次重绘闪烁。
- **提词器逻辑剥离**：移除 `TeleprompterView` 中直接渲染错误文案的代码，保持职责单一。
- **状态机自愈**：在 TTS 引擎下载成功或原生本地 TTS 降级朗读成功时，自动清除旧的错误状态 `_clearLastError()`。
- **吉祥物 (XIAOYO) 联动预埋**：`BoardMascot` 监听 TTS 错误，自动切换配色为警告粉色（`neonPink`）并展现难过/闭眼表情。

## 🚀 展望：V1.1 热更新与体验增强 (Next Steps)

- **XIAOYO 进化之路**：推进 Rive 骨骼动画集成，融入拟人化语音反馈（如 Cyber-Chirp 音效）。
- **自动化与热更新**：点亮 `DashboardScreen._checkAppUpdates()`，对接后端版本控制接口；支持云端动态下发 2048 皮肤。
- **数据监控与容错**：在 `CyberLogger` 中接入 Sentry 进行全域线上异常监控；优化 TTS 重试退避策略。

# 2026-04-23 模式匹配与状态机重构 (Switch Expressions)

## 🎯 今日任务目标

利用 Dart 3.11 最新的 Switch Expressions（穷尽式 switch 表达式）重构核心代码，消除静默失败风险并精简代码结构。

## 🛠 具体执行

### 1. UI 颜色映射重构 (`lib/shared/widgets/cyber_toast.dart`)
- **重构内容**：重构了 `_getBorderColor()` 方法。
- **重构方法**：将传统的 `switch-case` + `return` 语句改写为单行的 Switch Expression：
  `Color _getBorderColor() => switch (widget.type) { ... };`
- **重构收益**：去除了冗长的 `case`、`return` 和 `break` 语句，直接利用表达式返回值，代码更紧凑并获得了穷尽性检查。

### 2. 状态异常映射重构 (`lib/features/audio/services/tts_engine_service.dart`)
- **重构内容**：重构了 `_setLastError(dynamic error)` 方法中的异常信息映射逻辑。
- **重构方法**：引入了 Dart 3 的 Type checking patterns（类型匹配模式）和 Relational patterns（关系模式，如 `>= 500`），替代了原先冗长的 `if-else` 判断分支。
- **重构收益**：利用紧凑的解构语法实现了类型安全的模式匹配，极大提升了错误提示的声明式风格和可读性。

### 3. BackdropFilter 渲染性能调优 (Impeller)
- **扫描与裁剪约束**：审查了 `cyber_toast.dart`、`cyber_modal.dart` 以及 `privacy_agreement_modal.dart`。确认了前两者的模糊遮罩通过严格的嵌套结构规避了 Impeller 引擎下无边界渲染引发的 GPU 功耗飙升。
- **精准对齐**：确保了包含 `BackdropFilter` 节点内部背景 `Container` 的 `borderRadius` 同步使用 `CyberDimensions.radiusL` 约束，解决了边角模糊溢出边缘的抗锯齿隐患，彻底实现了视觉上的精确重合。

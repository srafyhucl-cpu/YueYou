## 3. Android 构建故障修复 (Kotlin 版本对齐)

### 问题现象

执行构建时报错：`e: Language version 1.6 is no longer supported; please, use version 1.8 or greater.`。故障点锁定在 `:sentry_flutter:compileDebugKotlin` 任务，原因为 Kotlin 2.2.0 编译器不再支持 1.6 版本的语言规范。

### 修复方案

- **全局对齐**：在根目录 `android/build.gradle` 中强制注入 Kotlin 编译选项，确保所有子项目（包括第三方插件）的 `languageVersion` 和 `apiVersion` 至少为 `1.8`。同时强制对齐 `jvmTarget` 与 Java `sourceCompatibility`/`targetCompatibility` 至 `17`。
- **架构健壮性**：合并多个 `subprojects` 配置块，并将 `evaluationDependsOn(":app")` 置于块末尾，规避 Gradle 提前评估子项目导致的配置时序冲突。
- **环境验证**：确保 Gradle 8.x 与 Kotlin 2.2.0 的兼容性配置正确。

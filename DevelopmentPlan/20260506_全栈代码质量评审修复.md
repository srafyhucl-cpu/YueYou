## 阶段二：性能与架构审计修复（下午）

### 性能 (高优)
- 移除 3 处 `build()` 中的 `CyberPerformanceDetector.detectLevel()` 调用（`cyber_toast.dart`、`cyber_modal.dart`、`teleprompter_view.dart`），改为从 `SettingsProvider.currentAnimationLevel` 读取缓存值（整个 session 只跑一次基准测试）
- 移除 `game_provider.dart` 中废弃的 `grid` 兼容层 getter，消除了不必要的矩阵复制

### Go 服务端加固
- `server/main.go`：新增 `http.Server` 配置 ReadTimeout/WriteTimeout/IdleTimeout；新增 SIGINT/SIGTERM 信号处理和 `srv.Shutdown()` 优雅关闭
- `server/handler_tts.go`：`exec.Command` → `exec.CommandContext(c.Request.Context())` 支持客户端断连时取消子进程；`os.Remove` → `defer os.Remove` 防止 panic 泄漏临时文件
- `server/response.go`：新建 `ok()` / `fail()` 统一响应格式，消除 handler 中重复的 `gin.H{"status":"error",...}` 样板
- 变量命名优化：`fn`→`objKey`、`fu`→`objURL`

### TTS 异步错误边界
- `tts_audio_notifier.dart` 中 4 个 `unawaited(Future.microtask(...))` 调用补 `.catchError` / try-catch + `CyberLogger.captureWarning`，防止 TTS 管线静默异常

### 新增测试
- `test/features/library/default_book_service_test.dart`：12 用例，覆盖目录后备方案、fetchChapter 边界、并发去重、缓存检查、预取

### 中优修复
- 空文件移除：`audio_controls.dart`、`game_screen.dart`（零行，无引用）
- `safeSubstring` 顶层函数 → `extension SafeStringExtension on String`
- `__CyberToastWidgetState` 双下划线 → `_CyberToastWidgetState`
- `cyber_player_console.dart` 混合导入 → 统一 package URI
- 头像 Mascot `_celebrate()` 增加 `if (!mounted) return;` 守卫
- `tts_cache_manager._listTtsFiles` 循环闭包变量提取 `final currentEntity = entity`
- `_executeDownload` 增加 try-catch 包裹下载 + 异常路径文件清理

### 代码去重
- `.txt` 后缀正则提取为 `TextProcessing.stripTxtSuffix()`，三处调用收敛
- WAV 文件头写入提取为 `AudioUtils.writeWavHeader()`，`ambient_service.dart` 和 `sfx_service.dart` 均删除了私有副本
- 语音白名单双向添加同步注释（`server/handler_tts.go` ↔ `settings_provider.dart`）
- `response.go` 的 `ok()`/`fail()` 统一了 handler 响应格式

### 空 catch 块注释
- 5 处关键路径空 catch 块添加了中文注释说明静默失败原因（dispose 路径、ping 失败、缓存清理）

### CI 去重
- `.github/workflows/flutter-ci.yml`：删除轻量 `build` job（与 `build-test` 重复）

# 阅游 (YueYou) 项目全域深度优化与重构指令集

**项目背景**：这是一个基于 Flutter 开发的「赛博朋克风格小说听读器 + 2048 游戏」APP。
**审计目标**：按照大厂级（阿里/腾讯/字节）工程、交互、容错标准，全面清剿代码库中的内存泄漏、状态竞争、UI 耦合与极端异常断点。
**执行要求**：请依次读取以下模块，严格按照我给定的「优化方案/策略」修改相关代码。涉及视觉修改时，**必须且只能**使用 `CyberDimensions`、`CyberColors` 和 `CyberTextStyles` 中的常量。

---

## 模块一：核心防呆与全域容错机制 (Critical Error Handling)

### 1.1 临时音频文件（MP3）泄漏（高危）

- **定位**：`lib/features/audio/services/tts_engine_service.dart`
- **问题**：异步清理策略在强杀 App 时失效，`tts_*.mp3` 长期遗留导致存储泄漏。
- **任务**：在 `TtsEngineService.init` 阶段，遍历 `getTemporaryDirectory()`，主动回收所有匹配 `tts_*.mp3` 且非当前 Session 的垃圾文件。

### 1.2 TTS 引擎与阅读器状态断裂（中危）

- **定位**：`lib/features/reader/providers/reader_provider.dart`
- **问题**：尚未导入书籍或书籍为空时，强行触发 TTS 会导致引擎崩溃或陷入无限异常循环。
- **任务**：在 `ReaderProvider.toggleTTS()` 及相关播放入口处增加前置拦截。如果 `_currentBookId == null` 或 `_sentences.isEmpty`，直接 return 并触发 UI 层的 Cyber 风格 Toast 提示（如：“未检测到有效数据块”）。

### 1.3 数据级联删除失效（高危）

- **定位**：`lib/features/library/providers/bookshelf_provider.dart`
- **问题**：`deleteBook` 仅删除了持久化数据，未重置内存中 `ReaderProvider` 的当前状态，导致切回首页后出现“幽灵章节”和越界崩溃。
- **任务**：重构删除逻辑。在 `deleteBook` 时，若删除的是当前正在阅读的书籍，必须同步重置 `ReaderProvider` 的状态（置空数据、停止 TTS 播放、归零进度）。

### 1.4 提词器（Teleprompter）字符越界崩溃（高危）

- **定位**：全局涉及 `substring` 或 `sublist` 调用的地方（特别是提词器高亮切片逻辑）。
- **问题**：直接操作 Index 切片极易因 Emoji 或特殊字符导致 `RangeError` 红屏。
- **任务**：提供并全局使用安全的截取工具函数：`String safeSubstring(String text, int start, int end)`，内部必须使用 `.clamp()` 限制索引不超出 `text.length`。

### 1.5 TTS 播放超时导致「静默跳句」（中危）

- **定位**：`lib/features/audio/services/tts_engine_service.dart`
- **问题**：`_startPlayLoop` 中 `_audioPlayer.setSource` 超时后直接 `continue`，用户会漏听句子且无感知。
- **任务**：超时后不应默默跳过。应暂停 TTS 播放循环，通过 `lastError` 抛出明确状态，让 UI 提示「音频加载超时，已暂停」，等待用户手动恢复。

## 模块二：内存、性能与工程架构 (Performance & Architecture)

### 2.1 TXT 导入的 OOM 与 Isolate 中断隐患（高危）

- **定位**：`lib/features/library/services/file_import_service.dart`
- **问题**：
  1. `await File(filePath).readAsBytes()` 直接将几十兆文件读入内存易导致低端机 OOM。
  2. 使用 `compute` 解析时，若用户按返回键退出，后台进程无法被终止（浪费 CPU，且易引发 Port Closed 崩溃）。
- **任务**：
  - 弃用简单的 `compute`，改用 `Isolate.spawn` 管理生命周期。
  - 在页面或组件的 `dispose` 生命周期中，提供 `cancelImport()` 方法并调用 `Isolate.kill()` 主动释放资源。

### 2.2 2048 游戏数据持久化写频与防抖（中危）

- **定位**：`lib/features/game_2048/providers/game_provider.dart`
- **问题**：高频滑动合并时高频触发 `_persistState()` 写入 SharedPreferences，引发微小卡顿。
- **任务**：引入 `Timer` 防抖机制（Debounce）。在用户滑动后 1 秒内无新操作时，或应用监听系统级 `AppLifecycleState.paused` 时，再执行实质的磁盘持久化。

### 2.3 UI 状态与领域模型 (Domain) 耦合（低危，架构债）

- **定位**：`lib/features/game_2048/providers/game_provider.dart`
- **问题**：Domain 内部维护了 `showGameOverDialog` 这种纯 UI 状态，不符合单向数据流。
- **任务**：剥离此变量。改用 `StreamController` 或事件回调，抛出 `GameOverEvent`，由 UI 层（`DashboardScreen` 等）监听并触发弹窗。

### 2.4 多 Provider 依赖带来的启动竞态（中危）

- **定位**：`lib/main.dart`
- **问题**：`SettingsProvider` 等异步读取本地缓存，如果 TTS 引擎在此之前初始化完毕，会读取到错误的默认音量/倍速。
- **任务**：构建一个显式的 Bootstrapper/Splash 屏流程。在渲染主页前，使用 `Future.wait` 确保所有的 `loadFromStorage` 完成。

## 模块三：视觉、交互与用户体验 (UI/UX & Accessibility)

### 3.1 弹窗组件原子化失效（高危 · 视觉规范必须 100% 落地）

- **定位**：`lib/shared/widgets/cyber_confirm_dialog.dart`
- **问题**：代码中出现了 `BoxConstraints(maxWidth: 360)`、`borderRadius: BorderRadius.circular(20)` 等大量硬编码，没有复用 `CyberDimensions`，与大厂规范背道而驰。
- **任务**：重构该组件。剥离底层重复的 `showGeneralDialog` 逻辑，强制将其作为 `showCyberModal` 的子节点 `child` 传入；所有边距、圆角、模糊值替换为 `CyberDimensions` 常量。

### 3.2 系统字体缩放导致的溢出截断（中危）

- **定位**：`lib/shared/widgets/cyber_confirm_dialog.dart` 及其他 Modal。
- **问题**：当系统设置字体放大 200% 时，被硬约束尺寸的容器会出现 Yellow/Black 溢出条。
- **任务**：所有弹窗的文本内容区（如 `message`）必须包裹在 `SingleChildScrollView` + `Flexible` 中，保证空间不足时可滑动。

### 3.3 异常状态恢复后的「僵尸弹窗」

- **定位**：全局弹窗 / 加载状态。
- **问题**：如果弹出 Loading 框时发生底层数据突变（如依赖的书籍被删、网络静默断开），`Navigator.pop` 可能永远不被调用。
- **任务**：在全局架构层面，推荐使用基于 Provider 监听的声明式 Overlay 控制加载状态（例如 `Consumer<GlobalState>` 决定是否显示 Loading），而非命令式的 push/pop 散落在异步流程中。

### 3.4 文本解析的「边缘截断」体验破坏（优化级）

- **定位**：`lib/features/reader/domain/text_parser.dart` (`_emergencySplit` 方法)
- **问题**：遇到超长无标点句，单纯按 50 字符截断极大概率会劈开词组（如把“赛博朋克”切成“赛博”/“朋克”），TTS 念出来极度怪异。
- **任务**：优化降级策略。在 50 字符阈值内寻找时，除了标点符号，增加对连词或语气助词（“的”、“了”、“和”、“与”）的探测作为软断点，提高 TTS 朗读的自然度。

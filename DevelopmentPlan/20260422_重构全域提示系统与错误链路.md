# 重构全域提示系统与错误链路

## 任务描述
全域提示系统重构与错误链路加固。主要涵盖 Toast UI 适配重构、ErrorListener 防抖加固、TTS 错误信息业务化映射，以及同步文档更新。

## 详细步骤

### 1. UI 适配与视觉重构 (CyberToast)
- **动态高度适配**：移除硬编码 `top: 40`，改用 `MediaQuery.of(context).padding.top + CyberDimensions.spacingM`。
- **视觉升级**：
  - 添加 `BackdropFilter` 实现 `sigma: 10` 的毛玻璃效果。
  - 边框应用 `CyberDimensions.borderThick`，圆角使用 `CyberDimensions.radiusL`。
  - 宽度采用 `IntrinsicWidth` 结合 `maxWidth` (屏幕宽度 85%) 支持长文本自动换行。
- **单例 Overlay 管理**：维护静态变量 `_currentEntry`，在显示新 Toast 前必须移除旧的 Entry 防止堆叠。

### 2. 逻辑链路加固 (TtsErrorListener)
- **防抖处理**：引入时间戳记录，对同一个错误信息执行 1000ms 防抖，避免重复弹出。
- **同步优化**：`tts.clearLastError()` 在 Toast 触发后立即执行，避免 Consumer 因 Widget 重绘反复触发。

### 3. 业务信息映射 (TtsEngineService)
- **错误脱敏与中文化**：收口 `_setLastError`。针对异常与网络错误代码实施中文化和业务化映射，避免直接暴露堆栈给用户。

### 4. 文档同步与提交
- 更新 `README.md`，同步组件调用变化（如有）。
- 更新 `development_log.md` 记录重构细节。
- 执行 `git commit` 并附上中文提交信息。

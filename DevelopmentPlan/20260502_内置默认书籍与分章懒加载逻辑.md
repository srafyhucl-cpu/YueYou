# 阅游任务单：内置默认书籍与分章懒加载逻辑 (20260502)

## 🎯 任务目标

实现《西游记》作为系统内置默认书籍的自动化加载逻辑。采用"目录先行 + 分章懒加载"方案，确保新用户"下载即看、切章即读"，并保持极高的性能表现。

---

## 📡 API 契约（与 TTS 分离下载原则完全对齐）

### 2.1 目录接口

```http
GET /book/catalog?bookId=xiyouji
Response:
{
  "status": "success",
  "bookId": "xiyouji",
  "title": "西游记",
  "totalChapters": 100,
  "chapters": [
    { "index": 0, "title": "第一回 灵根育孕源流出 心性修持大道生" },
    { "index": 1, "title": "第二回 悟彻菩提真妙理 断魔归本合元神" },
    ...
  ]
}
```

> **离线降级**：目录 100 条（仅标题，约 8 KB）作为 Dart 常量内置于 `BookConstants`，网络异常时直接使用内置目录，不影响书架展示。

### 2.2 章节文本接口（遵循分离下载原则）

#### 步骤一：POST 获取下载 URL

```http
POST /book/chapter
Body: { "bookId": "xiyouji", "chapterIndex": 0 }
Response: { "status": "success", "url": "https://cdn.example.com/books/xiyouji/001.txt" }
```

#### 步骤二：客户端 GET 下载纯文本

```http
GET https://cdn.example.com/books/xiyouji/001.txt
Response: 第一回正文纯文本（UTF-8，行分隔）
```

> 严禁直接用 POST 响应体返回章节文本，必须走 CDN URL 下载，与 TTS 契约完全一致。

## 🏗 架构设计与实现方案

### 1. 状态追踪 (Core/Storage)

#### 1.1 `IStorageService` 新增接口（必须先于实现）

```dart
// 粘性位：用户是否曾主动选择过书籍
bool hasSelectedBook();
Future<void> setHasSelectedBook(bool v);

// 分章文本缓存（存文件，非 SharedPreferences）
Future<void> saveChapterCache(String bookId, int chapterIndex, String text);
Future<String?> loadChapterCache(String bookId, int chapterIndex);
Future<void> clearChapterCache(String bookId); // 整本书清理
Future<void> pruneChapterCache(String bookId, {int keepAround = 3}); // LRU 清理

// 书目录缓存（存文件，与内置常量互补）
Future<void> saveBookCatalog(String bookId, List<Map<String, dynamic>> chapters);
Future<List<Map<String, dynamic>>?> loadBookCatalog(String bookId);
```

#### 1.2 `StorageService` 实现细节

- `user_has_selected_book`：使用 `SharedPreferences`，键名 `'user_has_selected_book'`。
- **分章缓存路径**：`{documentsDir}/books/chapters/{bookId}/{chapterIndex:03d}.txt`，**绝对禁止存入 SharedPreferences**（单章约 20–50 KB，SP 不适合大文本）。
- **目录缓存路径**：`{documentsDir}/books/catalogs/{bookId}_catalog.json`。
- `pruneChapterCache`：保留当前章 ±`keepAround` 范围内的章节文件，其余删除；防止 100 章全量缓存堆积（实际上 100 章 ≈ 1.6 MB，可接受，但仍需机制兜底）。

### 3. 分章懒加载服务 (Features/Library)

新增 `lib/features/library/services/default_book_service.dart`：

**职责边界**：纯网络 + 缓存服务，禁止引入 UI 库，不持有任何 Provider 引用。

```dart
class DefaultBookService {
  // 获取目录：优先本地缓存 → 网络拉取 → 降级内置常量
  Future<List<ChapterModel>> getCatalog();

  // 分离下载章节文本：优先命中本地缓存文件
  // 缺失时：POST 获取 URL → GET 下载文本 → 存入缓存
  Future<String?> fetchChapter(int chapterIndex);

  // 影子预读：fire-and-forget，不阻塞调用方
  void prefetchNextChapter(int currentChapterIndex);

  // 检查某章是否已有本地缓存
  Future<bool> isChapterCached(int chapterIndex);
}
```

### 4. `BookModel` 修复

**问题**：`toJson()` 未序列化 `chapters`，重启后书架恢复的 `BookModel` 无章节列表。

**方案**：对于默认书籍，章节目录通过 `StorageService.loadBookCatalog` 单独加载，**不存入书架元数据**（避免书架 JSON 膨胀）。`BookshelfProvider` 在 `loadFromStorage` 后需额外调用 `loadBookCatalog` 为默认书籍补充章节列表。

### 5. `ReaderProvider` 跨章状态机

**新增字段**：

```dart
int? _currentChapterIndex;           // 当前章节索引（默认书籍模式下有效）
ChapterLoadState _chapterLoadState = ChapterLoadState.idle;
bool _isDefaultBookMode = false;     // 是否处于分章懒加载模式
```

**新增枚举**（位于 `domain/` 层）：

```dart
enum ChapterLoadState { idle, loading, loaded, error }
```

**跨章边界处理流程**：

```text
TTS 播放到章末最后一句
        │
        ▼
onTtsItemFinished() → nextIdx >= _sentences.length
        │
        ▼
_autoAdvanceChapter()
  ├── _chapterLoadState = loading → notifyListeners() → UI 显示"正在同步云端卷宗..."
  ├── await DefaultBookService.fetchChapter(_currentChapterIndex! + 1)
  ├── 成功 → loadChapter(index+1, resume: false)
  │         → _chapterLoadState = loaded
  │         → _ttsNotifier?.refreshSession()  ← 替代"warmup"指令
  └── 失败 → _chapterLoadState = error → notifyListeners() → UI 显示重试提示
```

**新增方法**：

```dart
// 按章节索引加载（仅默认书籍模式）
Future<void> loadChapter(int chapterIndex, {bool resume = true});

// 章末自动推进（内部调用）
Future<void> _autoAdvanceChapter();
```

> **注意**：`warmup(5)` 不是现有方法，应替换为 `_ttsNotifier?.refreshSession()`，效果等价于重启预取泵从新章节开头开始填充缓冲队列。

### 6. 自动化链条 (Workflow)

在 `BookshelfProvider` 的 `loadFromStorage()` 或 App 启动时注入逻辑：

```text
App 启动
    │
    ▼
bookshelfProvider.loadFromStorage()
    │
    ├── 书架非空 → 正常恢复，结束
    │
    └── 书架为空 AND !StorageService.hasSelectedBook()
              │
              ▼
        DefaultBookService.getCatalog()  // 目录优先（常量降级兜底）
              │
              ▼
        BookshelfProvider.addDefaultBook(catalog)  // 注入元数据到书架
              │
              ▼
        DefaultBookService.fetchChapter(0)  // 静默下载第 1 章
              │
              ├── 成功 → ReaderProvider.loadChapter(0)  // 预热阅读器
              └── 失败 → 静默，用户进入后手动触发
```

**边缘场景处理**：

| 场景 | 处理策略 |
| --- | --- |
| 老用户删光书籍回到空书架 | `hasSelectedBook() == true`，不触发自动注入 |
| 下载第 1 章时无网络 | 静默失败，阅读器显示"无内容，点击重试" |
| 用户已在阅读其他书籍时 App 启动 | `currentNovelId` 非 `xiyouji`，跳过自动注入 |
| 影子预读失败 | 仅打印警告，不影响当前章节阅读 |

## ⚠️ 风险提示

- **索引对齐**：`BookConstants` 内置标题的顺序索引必须与后端 CDN 章节文件命名（`001.txt` 起始为第 0 章）完全一致，任何偏移都将导致章节内容错乱。
- **内存管理**：`pruneChapterCache` 需在每次成功下载新章节后调用，防止极端情况下积累过多缓存文件。
- **并发防护**：`fetchChapter` 必须持有章节级别的锁（或 `Completer` 缓存），防止同一章节被同时发起两次下载请求。
- **IStorageService 优先**：所有新增存储方法必须先更新接口，再更新 `StorageService` 实现，确保测试层 Mock 覆盖。

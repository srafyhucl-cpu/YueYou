# 阅游任务管家

## 触发场景

在阅游项目中，只要一次开发任务进入收尾阶段，必须使用本技能检查交付闭环。

## 收口清单

1. 运行必要验证：优先 `flutter analyze`，按风险补充目标测试或全量测试。
2. 检查今日任务文件：`DevelopmentPlan/YYYYMMDD_中文概要.md`。
3. 同一天只保留一个今日任务汇总文件。若已有今日文件，应更新它，不再新建第二个。
4. 任务完成后更新 `development_log.md`，记录日期、类型、核心变更、验证结果。
5. 只有当用户可见能力、启动方式、依赖、架构事实发生变化时，才更新 `README.md`。
6. 使用中文 Conventional Commit 提交，例如 `feat: 接入环境音播放服务`、`fix: 修复 TTS 会话竞态`。
7. 提交后按用户项目约定推送当前远程分支。网络或权限受限时，请求授权后再执行。

## 发布前协作

- 如果任务涉及打包、发版、上线、版本号、生产环境配置或交付测试包，必须同时使用 `yueyou-release-readiness-guard`。
- 若只是普通代码任务收口，仍以本技能为主，但需要确认是否触发发布守卫。
- 发布守卫负责发版门禁，本技能负责文档、提交、推送和每日任务文件闭环。

## 文档规则

- Markdown 必须使用 UTF-8。
- 标题、表格、列表必须能被常见 Markdown linter 正常解析。
- 不写空泛复盘，记录可追溯的代码路径、行为变化、验证命令。
- 不因为收口而重写无关历史文档。

## Git 规则

- 提交前先看 `git status --short`，区分本次修改和用户已有修改。
- 只暂存与本次任务相关的文件。
- 禁止回滚用户改动。
- 禁止批量删除文件或目录。

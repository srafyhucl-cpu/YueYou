# PROD-07 匿名商业证据计算器

## 当前状态

这是一个只处理匿名聚合计数的离线计算工具，不是支付实现，不创建订单、权益、账号或
购买网关。当前没有真实支付台账输入，因此不能据此宣称商业实验已启动或任何指标成立。

工具入口：

```powershell
dart run scripts/evaluate_commerce_experiment.dart <匿名聚合 JSON 路径>
```

输入必须使用 `schemaVersion: 1`，并只包含实验前置条件、匿名人数计数、分币支付/退款
金额和云音色合规证据。缺失字段会输出 `insufficientEvidence`；计数超过分母、退款超过
总支付或出现正文/书名/路径/账号字段时直接拒绝输入。

## 固定计算范围

工具按照详设第 21.2 节计算：

- Local Pro 订金率、全款率和 79 元以上支付人数。
- 七日回访率、书境包全款率、Xiaoyo 前三付费理由比例。
- 作者听校全款率、云音色持牌/隐私/毛利/单位经济状态。
- 总支付、退款和净支付，单位为分；退款不会被隐藏在收入中。

单项状态为 `continueLine`、`adjustLine`、`stopLine` 或 `insufficientEvidence`。总体状态
只有在支付渠道、退款演练和负责人批准均为 `true` 后才可能进入指标决策；前置条件为
`false` 时为 `notStarted`，缺失时为 `insufficientEvidence`。Local Pro 与书境包同时
触发停止线时，才触发普通用户商业系统总停止线。

## 隐私边界

输入不得包含用户 ID、账号、手机号、邮箱、设备 ID、书名、正文、章节文本、游标、TTS
文本或路径。工具只输出比例、计数、金额和决策原因，不保存用户级实验记录。

## 当前结论

当前没有真实商户订单、退款演练、D0/D7 台账、付款后访谈或云音色供应商证据；因此
`PROD-07` 仍保持“未启动真实支付实验”，不允许用测试 fixture、问卷或模拟点击替代真实
付款，不进入正式 `commerce` 实现。
